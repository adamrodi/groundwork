# Invoices

## What it does

Salim can view invoices, download them as PDF, send them via SMS (iMessage deep link) or email (Resend Edge Function), and mark them paid. Clients view invoices publicly at `/i/:id` without logging in.

## Key files

- `src/pages/Invoices.tsx` — List view with number, client, total, status, date.
- `src/pages/InvoiceDetail.tsx` — Detail with actions: Download PDF, Copy Link, Send SMS, Send Email, Mark Paid. Actions vary by status.
- `src/pages/InvoicePublic.tsx` — Client-facing public view (no auth required). Also hosts the Stripe payment form.
- `src/components/InvoicePDF.tsx` — Client-side PDF via `@react-pdf/renderer`. `downloadInvoicePDF()` triggers browser download.
- `supabase/functions/send-invoice/index.ts` — Edge Function that sends invoice email via Resend API.

## Architecture decisions

- **Client-side PDF generation** — No server needed. `@react-pdf/renderer` converts React components to PDF. Bundle is ~1.2MB but acceptable for MVP.
- **Anonymous RLS policy on invoices** — `/i/:id` must work without auth. UUID v4 (122 bits of entropy) serves as the access token. Post-MVP: add signed tokens.
- **SMS uses iMessage deep link** — `sms:phone?body=url` opens iMessage prefilled. No backend needed. Status optimistically updated to `sent`.
- **Three invoice states: draft → sent → paid** — Linear status flow. Only valid transitions: draft→sent (on send), sent→paid (on payment/manual mark).

## Deploy

```bash
supabase functions deploy send-invoice
```

Requires secrets: `RESEND_API_KEY`, `APP_URL`.
Client env: `VITE_APP_URL` (base URL for public invoice links).

## Gotchas

- Invoice number is auto-generated by Postgres SERIAL. Never set manually.
- SMS status update is optimistic — no way to detect if user actually sent the iMessage.
- Resend `from` address must be verified. Use `onboarding@resend.dev` for testing.
- InvoicePDF must be mocked in tests (canvas APIs unavailable in jsdom).
- Email Edge Function uses service role key (reads invoice with full client data). No explicit auth needed — it runs server-side.
- `VITE_APP_URL` must match the Vercel deployment URL or public links break.
